Investigación

URL: Uniform Resource Locator Es una secuencia de carácteres que sigue un estándar, éste identifica y localiza recursos e información determinada en Internet.
MIME: Multipurpose Internet Mail Extensions, una serie de convenciones para el intercambio de archivos, enviados a través de SMTP (Simple Mail Transfer Protocol ), por lo que los mensajes de correo electrónico son enviados a través de estos 2 formatos.
Métodos HTTP: Son obligatorios para la realización de solicitudes a un servidor usando el protocolo HTTP, algunos de los métodos son: GET, HEAD, POST, PUT, PATCH, DELETE.
Códigos de errores HTTP: Son usados para indicar el estado de las solicitudes hechas a un servidor HTTP, las respuestas se dividen en 5 clases: informativas, satisfactorias, redirecciones, errores de los clientes y errores de los servidores.
XML: Extensive Markup Language, es un meta-lenguaje por lo que no es un lenguaje de marcado, éste permite definir lenguajes de marcado adecuados a W3C, es un subconjunto de SGML(Stándar Generalised Mark-up Language).
XHTML: eXtensible HyperText Markup Language, basado en HTML 4 que se caracteriza por ser más estricta y limpia y la capacidad de combinar HTML para mostrar datos y XML para describir datos, permite una mejor interpretación de la información que HTML.
HTML5: HyperText Markup Language, versión 5, es la quinta versión de HTML, se caracteriza por rediseñar el código para resolver problemas, actualizaciones para nuevas necesidades, características nuevas y plataformas de desarrollo de aplicaciones web complejas.
JS: JavaScript, es el lenguaje de programación de la web, es interpretado y orientado a objetos, tiene capacidades dinámicas como construcción de objetos en tiempo de ejecución, listas variables de parámetros, variables que pueden contener funciones,etc.
php: Hypertext Preprocessor ,Es un lenguaje adecuado para el desarrollo web que puede ser incrustado en HTML, el código es ejecutado en el servidor, su meta es permitir la creación de páginas web generadas dinámicamente.
jsp: JavaServer Pages, su propósito es crear páginas web dinámicas basadas en HTML y XML, es similar a PHP pero jsp está basado en Java.
Ajax: Asynchronous JavaScript And XML, es una técnica de desarrollo web para la creación de aplicaciones interactivas mejor conocidas como RIA (Rich Internet Applications) las cuales se ejecutan en el cliente mientras mantiene una comunicación asíncrona con el servidor esto quiere decir que los datos solicitados en el servidor, se cargan en segundo plano sin interferir con la visualización y comportamiento de la página.

¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?
GET: Usado para adquirir un archivo o recurso en uns servidor web
HEAD: Similar a GET sólo que solicita los metadatos de un recurso y no de todo el elemento.
POST: Usado para enviar información al servidor y que sea almacenado como subelemento de un recurso ya existente
PUT: Crea, carga o reemplaza un recurso en un servidor.
PATCH: Aplica modificaciones a un recurso.
DELETE: Borra un recurso en un servidor web.

¿Cuáles son las diferencias principales entre HTML 4 y HTML5?

  • HTML5 requiere del elemento DOCTYPE al principio de la página.
  • En HTML5 los elemento < div > y < li > pueden contener contenido de elementos block o inline pero no los 2, cosa que era un bug en HTML4
  • HTML5 tiene nuevos elementos como: < article >, < aside >, < dialog >, < footer >, < header >, etc.
  • HTML5 agrego también nuevos atributos para mejorar la consistencia de elementos ya existentes en HTML4 como: media, ping, target e inlcuso atributos globales.
  • Se han eliminado elementos como: acronym, applet, dir, center, big, etc.
  • También se han eliminado atributos.
